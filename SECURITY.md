# Security policy

## Supported versions

| Version | Supported |
|---------|-----------|
| `0.x` (current) | Latest commit on `main` — best effort |

## Reporting a vulnerability

Please **do not** open public issues for undisclosed security vulnerabilities.

Preferred: use **[GitHub private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing/privately-reporting-a-security-vulnerability)** for this repository (if enabled by the org), or contact repository maintainers through a private channel they publish on the GitHub org or user profile.

Include:

- Description of the issue and impact
- Steps to reproduce (proof-of-concept if possible)
- Affected versions or commit SHA

Maintainers will acknowledge receipt when possible and coordinate disclosure after a fix.

## Scope

This policy covers the **stratum-map** library, examples, and scripts in this repository. Third-party dependencies (OpenLayers in demos, Playwright, etc.) follow their own upstream security policies.
